Will the minicoredumper work with stripped libraries and executables?

There are some symbols that minicoredumper will need to find. These should not be stripped from their libraries and executables.

  1. Any symbols specified as buffers in the recept file must not be stripped.
  2. The folling symbols within the libminicoredumper library must not be stripped:
    • mcd_dump_data_version
    • mcd_dump_data_head
  3. Although normally not needed, minicoredumper will fallback to manually check for some symbols in libpthread if the crashing application is using a libpthread that is incompatible with the libthread_db minicoredumper is using. Those libpthread symbols are:
    • _thread_db_sizeof_pthread
    • stack_used
    • __stack_user
