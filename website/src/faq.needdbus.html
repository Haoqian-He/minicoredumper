Is D-Bus support necessary?

D-Bus is used by minicoredumper to trigger libminicoredumper-based applications to dump their data while running. If there will be no libminicoredumper-based applications running on the system or the trigger functionality is not needed, there is no reason to enable D-Bus support. D-Bus support can be disabled by providing --without-dbus as a command line argument when running the configure script.
