How can I prevent static data from being stripped from the core file?

Using the tool nm on the library or executable containing the symbol, it is possible to see the name of the symbol as used by the linker. This name can be specified as one of the buffers of a recept file.

As an alternative, if the application is using libminicoredumper, it can register specific data for dumping with mcd_dump_data_register_bin(). However, this method may not always be appropriate.
